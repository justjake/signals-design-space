# fx2 — React updater queues carry worlds

## How it works
Canonical state is a conventional signal graph (graph.ts); concurrency is a replay overlay (worlds.ts). A transition's writes become a draft — an ordered intent log — and any cell a draft touches keeps a rebase log: base value plus all intents in global dispatch order. Canonical readers replay urgent intents only; a world replays canonical plus its drafts' intents in sequence, so functional updates rebase correctly ((1+2)*2=6) with no lane bookkeeping. Visibility is React state: each root's SignalScope holds a useReducer of draft ids dispatched inside the owning startTransition, so React's own update queues decide which passes see which drafts — urgent passes skip, transition passes include, rebased retries recompute; a revision counter re-dispatches late appends. useValue's useSyncExternalStore snapshot is a subscription epoch, never a value, so drafts never read as store mutations (no sync-fallback de-opt); the render value resolves the pass's world from context. Writes classify by reading React's live transition object (internals .T, WeakMap-keyed). Retirement folds intents through the ordinary write path once every recipient root commits. The 11 fork lines only bracket the commit mutation phase (__FX2_MUTATION_WINDOW__ start/stop) plus a load-time protocol marker.

## Advantage
Rebase, replay-on-interrupt, urgent-during-transition arithmetic, transition holding, time slicing, and StrictMode netting are inherited from React's updater machinery rather than reimplemented — the incumbent pays 1510 fork lines for what fx2 gets from 11 (137x), with library 2239 vs ~4700–5000. Per-episode structures (drafts, rebase logs) drop at quiescence, so the steady-state engine is a plain signal graph. The epoch snapshot dodges both the transition de-opt and the post-commit repair storm with zero fork support.

## Disadvantage
Raw graph speed: CI geomean 1.327x alien (self-measured 1.278), worst case unstable 2.96x; reads under live drafts resolve through rebase logs. Correctness rides subtle epoch/scope/fold choreography — both judged defects were exactly this class, suggesting more edges exist. Classification reads React internals (.T/.H): read-only but upgrade-sensitive. Seam overhead: ~18% on fanout/mount vs a bare uSES store, 10ms p95 urgent during transitions. The global __FX2_MUTATION_WINDOW__ slot is last-writer-wins under duplicate bundles.

## Room for optimization
The gap is engine-side, not seam-side: the mark wave re-walks subscriber lists per write (propagation family 1.2–1.5x) and dynamic-dependency churn re-links eagerly (unstable 2.96x). Levers: (1) alien-style two-generation mark — REPORT expects it to close most of the 1.3–1.5x band; (2) batching uSES notification fan-out per commit to recover the ~0.5ms fanout delta and ~18% mount overhead; (3) a per-pass world beacon (SignalScope re-rendering atop urgent passes) closes the residual latest() corner without new fork lines.

## Bugginess
Judging found two undisclosed gaps — bare-root subscribers stale after silent folds; useIsPending not flipping during transition-owned refresh — both fixed with pre-fix-failing regressions plus new oracle coverage of the fold class; final verdict clean. Gates: 179/179 conformance, 224/224 engine, 31/31 real-React, 25/25 shared battery, 1200-seed oracle with sabotage canaries, 6/6 leak audit, bit-identical patch replay. Residual: the disclosed latest()-without-hooks edge and the duplicate-bundle carrier. High confidence inside documented shapes (wrapCreateRoot).
