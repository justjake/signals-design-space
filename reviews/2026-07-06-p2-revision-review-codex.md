## Declared retreats

1. **Dead-arena lane loss — acceptable retreat.** At U’s write, W’s committed view still renders `flag=false`, hence still reads `b`; U alone does not change what W rendered. SP5 therefore does not require delivery yet. If T becomes durable first, site (b) retracks to `a` and U’s later retirement corrects; if U retires first, `flag=false` still hides it and T’s later lock-in/retirement refold observes U’s now-permanent value. SP4 applies only to work actually scheduled, so it is not violated when U schedules none ([SP4/SP5](/Users/jitl/src/alien-signals-opt/spec/react-compliance-contract.md:494)). The engine⊆model delivery tolerance merely permits the omission; the exact snapshots/correction stream must prove the value floor ([comparator](/Users/jitl/src/alien-signals-opt/packages/cosignal/tests/helpers.ts:143)). This is a real responsiveness loss versus HEAD, but not disguised unsoundness.

2. **World-read fast-path deletion — acceptable retreat, but its gate is staged incorrectly.** Removing cache borrowing is semantically clean. The revision deletes it in S-A ([S-A](/Users/jitl/src/alien-signals-opt/plans/2026-07-06-effects-unification-and-nf2.md:1065)), while the full bench battery is deferred to S-D ([S-D](/Users/jitl/src/alien-signals-opt/plans/2026-07-06-effects-unification-and-nf2.md:1111)) and has no numeric acceptance threshold. The cold-pass result must gate the S-A deletion, not merely diagnose it three stages later. This is a staging/performance defect, not a correctness retreat.

3. **Permanent many-root residual — disguised resource unsoundness.** “R arenas × 32–64 KB” assumes R means live roots and that 32–64 KB is a bound. Neither holds. The fork protocol has no root-destroy notification ([surface](/Users/jitl/src/alien-signals-opt/packages/cosignal-react/types/react-fork.d.ts:11)); shim root maps are never pruned ([root creation](/Users/jitl/src/alien-signals-opt/packages/cosignal-react/src/shim.ts:283)); and shim disposal disconnects callbacks without clearing bridge roots or engine state ([dispose](/Users/jitl/src/alien-signals-opt/packages/cosignal-react/src/shim.ts:244)). The spike’s arena grows geometrically and only releases its map/value columns on whole-world discard ([growth](/Users/jitl/src/alien-signals-opt/research/experiments/world-tagged-links-spike-code/cosignal/src/index.ts:2283), [discard](/Users/jitl/src/alien-signals-opt/research/experiments/world-tagged-links-spike-code/cosignal/src/index.ts:2783)). Under the revision, R therefore means roots ever materialized, and each keeps its high-water buffer, links, values, and outcome boxes through quiescence.

## New findings, ranked

1. **BLOCKER — the settlement hook is below the event it must observe.** The plan says every `ctx.use` settlement already calls `invalidateComputed`; HEAD does not. `unwrapThenable`’s shared listener only updates `status/value/reason` ([HEAD](/Users/jitl/src/alien-signals-opt/packages/cosignal/src/index.ts:1641)). The computed-specific listener is installed only when the **kernel** `storeThrown` caches that thenable, and it calls `invalidateComputed` only after a kernel-slot stale guard passes ([HEAD](/Users/jitl/src/alien-signals-opt/packages/cosignal/src/index.ts:1769)). Walk: committed R evaluates C with world key A and caches pending A in its arena; newest evaluates C with key B and holds a settled value; A settles. Kernel C neither caches A nor passes the stale guard, so `invalidateComputed(C)` never runs and the bridge hook never fires. R’s clean sentinel remains. Fixing this requires notification at thenable settlement plus distinct-thenable/computed dedup; “one closure per bridge, zero per-key/per-arena allocation” has not supplied that mechanism.

2. **BLOCKER — site-(b) refold-always has no representation.** Dirty marks and lists store only node ids/flags ([lifetime row](/Users/jitl/src/alien-signals-opt/plans/2026-07-06-effects-unification-and-nf2.md:1008)); consumption occurs later during evaluation. Nothing carries “this mark came from lock-in, never trust fp.” The available approaches are eager site-(b) refold, a sticky per-shadow force-refold bit, or dropping fp gating and always refolding marked atoms. They have different commit-path costs and must be chosen explicitly. This also means B3’s original seq-50-under-100 pin currently fails the written mechanism.

3. **BLOCKER — one weak bit does not define mixed tracked/untracked dependency semantics.** One `(dep,sub)` link may be read both ways in one evaluation and may change mode between evaluations. Walk: C reads A tracked, then A untracked. If the last read overwrites the flag, A’s write skips C despite the tracked occurrence; if first-mode wins, a later untracked→tracked evaluation can leave the link weak. The reuse algorithm must reset mode on the first occurrence of each evaluation, make strong dominate duplicates, and update mode on reused links. Observation capture must include only the strong occurrence; “every dependency read” currently risks making an untracked atom participate in OL1. The spike’s `wLink` returns early on exact tail reuse and carries no mode logic ([prototype](/Users/jitl/src/alien-signals-opt/research/experiments/world-tagged-links-spike-code/cosignal/src/index.ts:2332)).

4. **BLOCKER — permanent committed arenas have neither root death nor shadow reclamation.** After one consumer populates an arena and unmounts, no specified operation destroys its top-level shadows, links, cached references, or arena. Quiescence deliberately preserves all of it. Repeating this with fresh root containers yields unbounded memory and makes every fanout/settlement scan O(roots ever seen). Read-clock “renumbering” additionally zeroes stamps across all permanent arenas at quiescence ([§4.5.7](/Users/jitl/src/alien-signals-opt/plans/2026-07-06-effects-unification-and-nf2.md:984)), so the claim that quiescence “sheds work” is false once dead-root high-water arenas accumulate.

5. **BLOCKER / contract stop — the lifetime table still uses lifetimes outside the mandatory four.** The contract requires exactly one of L1/L2/L3/L4 before choosing the structure ([rule](/Users/jitl/src/alien-signals-opt/spec/react-compliance-contract.md:99)). The table uses “L1-derived,” “same lifetime as their arena,” “mechanism,” and “node lifetime” for committed arenas, marks, pools, hooks, and rawFn/equality columns ([table](/Users/jitl/src/alien-signals-opt/plans/2026-07-06-effects-unification-and-nf2.md:1004)). Some instances can be expanded to L1 or L3, but the pool, bridge hook, and node policy state require either exact classification or an owner amendment saying infrastructure state is outside the four-lifetime rule.

6. **MAJOR — weak links move untracked cost onto every write.** HEAD keeps weak adjacency separate, so delivery never scans it ([HEAD weak table](/Users/jitl/src/alien-signals-opt/packages/cosignal/src/logged.ts:2056)). A combined arena subscriber list must visit and bit-test every weak edge merely to skip it. A hot atom with K untracked-only dependents across R arenas becomes O(RK) per write instead of zero weak-edge delivery work. The current bench battery has no untracked-fan shape.

7. **MAJOR — lock-in refold cost and atom fingerprints are not honestly priced.** Site (b) scans each locked token’s `atomsTouched`, marks the root arena, and must eventually refold every consumed marked atom without fp rescue. Interleaved writes can put duplicate atoms in `atomsTouched` at HEAD ([append rule](/Users/jitl/src/alien-signals-opt/packages/cosignal/src/logged.ts:2474)). Moreover, fresh marks from sites (a), (c), and (d) always advance retirement/base/write maxima, while site (b) may never fp-gate. Thus the per-arena atom fp has no demonstrated successful skip site and can add a fingerprint tape scan before the inevitable fold. A wide-token/below-max lock-in benchmark is missing.

8. **MAJOR — S-A is honest but no longer isolating.** One stage introduces allocation, strong/weak link modes, folds, all four flip sites, mark persistence, settlement, equality, boxes, committed-arena permanence, and fast-path deletion before removing the memo arms. A divergence only narrows to “most of the new mechanism.” More importantly, its test-first list lacks the world-only settlement schedule, mixed strong/weak reuse, site-(b) mark provenance, and root-churn retention schedules above. The cold-pass performance check also arrives only at S-D.
