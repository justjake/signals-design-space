title: Devtools and tracing
description: Inspect the live dependency graph and follow events back to the write that caused them.
---

Every playground on this site has an **Open devtools** button. Interact with
the example first, then open its graph and causal event log. All playgrounds on
the page share one collector, matching how one application connects one engine.

```tsx
const { collector } = attachFx2Devtools()

createRoot(panel).render(<DevtoolsPanel backend={collector} />)
```

<Playground name="counter" />

The lower-level `signals-royale-fx2/debug` entry point exposes the same inert
inspection contract used by the panel. `inspect`, `deps`, and `subs` read stored
node fields without evaluating a stale computed, so debugging does not change
the graph it observes.

For a lightweight text trace, `attachTracer` stores a bounded ring of causal
events. `whyLastDelivery(node)` walks from a component or effect delivery back
to the write, settlement, or transition retirement that caused it. Stop the
tracer when the session ends; only one tracer or devtools sink is active at a
time.
