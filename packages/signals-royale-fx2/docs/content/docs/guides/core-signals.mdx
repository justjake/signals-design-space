## Atoms and computeds

`createAtom` makes writable state. `createComputed` is lazy and cached: it
re-evaluates only when read after one of the signals used by its previous
evaluation changed.

Use `set` to replace an atom value and `update` when the next value depends on
the previous one. The methods on an atom and the exported functions have the
same behavior.

## Reducer atoms

A reducer atom fixes one reducer at creation. Its `dispatch` method records the
reducer application during a transition, so it can replay over urgent writes
that arrive before the transition commits.

```tsx
const counter = reducerAtom(
	(state: number, action: 'increment' | 'reset') =>
		action === 'increment' ? state + 1 : 0,
	0,
)

counter.dispatch('increment')
counter.dispatch('reset')
```

Keep reducers and updater functions pure. React can replay them when concurrent
work restarts.

## Batching and equality

Every atom defaults to `Object.is` equality. An equal write stops before graph
propagation. `ComputedOptions.equals` and `EffectOptions.equals` provide the
same cutoff for derived values and effect delivery.

`batch` delays propagation until the callback closes. Reads and writes inside
the callback still happen in order, but downstream work settles once.

```tsx
batch(() => {
	width.update((n) => n + 1)
	height.update((n) => n + 1)
})
```

<Playground name="batch-effect" />

`startBatch` and `endBatch` expose the same mechanism when work cannot fit in
one callback. Every `startBatch` must have a matching `endBatch`, including on
errors, so prefer `batch` when possible.

## Reading deliberately

- `signal.get()` and `read(signal)` track a dependency inside a computed or
  effect source.
- `signal.peek()` reads the same snapshot without tracking it.
- `untracked(fn)` suppresses tracking for all reads in the callback.
- `latest(signal)` reads the newest view available to the current context and
  never suspends.
- `isPending(signal)` reports newer draft or async data behind the value being
  shown without evaluating the signal.

Inside a render or computed, `latest` stays within that evaluation's snapshot.
Reading another transition's draft would mix two possible futures and tear.
