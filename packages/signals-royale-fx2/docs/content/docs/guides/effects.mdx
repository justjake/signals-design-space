---
title: Effects
description: React to settled signal values without mixing tracking and side effects.
---

An effect separates what it watches from what it does. The source runs tracked;
the handler runs untracked. A signal read that should trigger the effect belongs
in the source, not the handler.

```ts
const stop = effect(
	() => total.get(),
	(value, previous) => console.log({ value, previous }),
)

stop()
```

The source can also be one signal, a tuple of signals, or a record of signals.
Tuple and record sources default to `shallowEquals`, because they create a new
container for each evaluation.

The handler runs once at creation when the source settles. It may return a
cleanup, which runs before the next handler and at disposal. A pending source
stays silent; a rejected source throws from the lane that drains it.

## React-owned effects

`useSignalEffect` and `useSignalLayoutEffect` own an engine effect for one
component. Their factory follows React's dependency-array lifecycle, while
signal-triggered handler runs use the matching passive or layout phase.

```tsx
useSignalEffect(
	() => ({
		watch: { width, height },
		run: ({ width, height }) => analytics.dimensions(width, height),
		label: 'dimensions',
	}),
	[width, height],
)
```

Use `effectScope` outside React when several effects share one lifetime. The
returned disposer releases every effect created in the scope, including nested
effects created by a handler.
